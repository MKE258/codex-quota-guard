# 项目额度守卫

用于控制一个本地项目的每日额度。核心公式：

```text
今日建议额度 = 当天开始时的剩余额度 / 距离刷新剩余天数
```

## 安装版使用方法

1. 双击 `CodexQuotaGuard-Setup-v1.1.2.exe`。
2. 完成安装后，从桌面或开始菜单打开 `Codex Quota Guard`。
3. 点击“登录 Codex 网页”，登录自己的 ChatGPT 账号。
4. 开启自动同步。

换电脑后不需要复制旧电脑的登录目录。重新登录更安全。

## 免安装版使用方法

1. 解压压缩包，双击 `CodexQuotaGuard.exe`。
2. 点击“登录 Codex 网页”，在弹出的独立浏览器中登录 ChatGPT。
3. 确认浏览器显示 Codex Usage 页面后，关闭浏览器窗口。
4. 在工具中点击“立即同步网页额度”。
5. 填写项目启动命令，例如 `npm run dev` 或 `python app.py`。
6. 选择项目目录，点击“保存设置”。
7. 勾选“自动同步 Codex 网页额度”，选择同步间隔，点击“启动项目”。

如果只想查看额度，请保持“仅监控额度”勾选。此时不需要填写项目命令和目录，也不会启动或停止本地项目。

如果不想在工具专用浏览器里重新登录，可以勾选“使用系统 Chrome 登录状态”。此模式会使用当前 Windows 用户的普通 Chrome 登录目录；同步前需要关闭所有普通 Chrome 窗口，否则 Chrome 可能因登录目录正在使用而拒绝读取。

## 模式和策略

- `仅监控额度`：只查看、同步和提醒，不管理本地项目。新用户默认使用此模式。
- `项目守卫`：取消勾选“仅监控额度”后，可填写项目命令和目录。
- `仅提醒`：达到今日建议额度后只弹出提醒，不停止项目。默认使用此策略。
- `提醒后停止`：提醒后停止由工具启动的项目。
- `立即停止`：达到上限后直接停止由工具启动的项目。

自动同步间隔可以选择 `5 / 15 / 30 / 60` 分钟。界面会显示最后同步时间和距离下次自动同步的时间。

额度页面地址为 `https://chatgpt.com/codex/settings/usage`。默认浏览器登录状态保存在当前 Windows 用户的 `%LOCALAPPDATA%\CodexQuotaGuard\browser-profile` 目录中，不会保存密码。应用设置保存在 `%LOCALAPPDATA%\CodexQuotaGuard\quota_guard_state.json`。

达到当天建议额度后，工具会自动结束由它启动的项目进程树。网页无法读取时，也可以手动填写剩余额度、刷新时间，并登记消耗量。

首次运行时，Windows 可能提示这是来自互联网的未签名应用。公开版目前没有代码签名证书，请仅从可信来源下载，并在运行前核对发布者提供的 SHA-256。

## 当前限制

Codex 套餐额度没有公开 API，因此自动同步依赖 Usage 网页。网页结构发生变化后，读取脚本可能需要调整。自动同步间隔为 `5` 分钟，因此停止项目最多可能存在约 `5` 分钟延迟。

自动同步时会在屏幕外短暂启动 Chrome 或 Edge，读取完成后自动关闭。这是为了兼容 ChatGPT Usage 页面的加载方式，不会弹出到桌面中央。

## 隐私说明

- 工具不是 OpenAI 官方产品。
- 工具不会要求输入或保存 ChatGPT 密码。
- 登录状态仅保存在本机 `%LOCALAPPDATA%\CodexQuotaGuard\browser-profile`。
- 勾选“使用系统 Chrome 登录状态”时，工具会让 Chrome 使用普通 Chrome 用户目录读取网页，不会复制 Cookie。
- 应用设置仅保存在本机 `%LOCALAPPDATA%\CodexQuotaGuard\quota_guard_state.json`。
- 不要把该浏览器目录发送给其他人。
